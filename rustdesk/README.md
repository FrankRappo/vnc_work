и публичный сервер RustDesk (теперь блокирует коннекты без логина — анти-абьюз).

## Зачем свой сервер
Публичный сервер RustDesk (`rs-*.rustdesk.com`) с 2026 требует вход (Google/GitHub) перед
коннектом и прямо пишет «не для продакшна». Поэтому подняли собственный hbbs+hbbr на VPS —
без логин-стены, с рабочим буфером обмена и без лимитов по времени.

## 🔴 Важные грабли
- **Песочница режет долгие процессы сигналом 16 (exit 144).** Все команды канала (rustdesk/anydesk/Xvfb на :99,
  ssh к VPS долгий) запускать с `dangerouslyDisableSandbox`.
- **Self-match pkill:** не писать в командах строки `Xvfb :99` / `rustdesk_kso.sh up` рядом с pkill — скрипты
  внутри делают `pkill -f "Xvfb :99"` и убивают сам шелл. Глушить сирот по явному PID.
- **Сироты `up`:** старый драйвер `rustdesk_kso.sh up` зависает и не выходит → копятся дубли, рвущие коннект
  (`kill_on_display rustdesk` по кругу). Здесь используем GUI-подход (connect.sh), без голого `--connect`.
- **Раскладка кассы:** ввод латиницы/спецсимволов прямо НА кассе (через канал) может уехать в кириллицу —
  проверять зумом. Ввод в НАШИ локальные диалоги (:99) безопасен (раскладка US).

## Клавиша Super/Win НЕ пробрасывается (и это НЕ нужно чинить)
Симптом: `xdotool key super+r` на :99 НЕ открывает «Выполнить» на кассе — Win-комбо не доходят.
Причина: локальный WM (fluxbox) и X-сервер на :99 перехватывают Super раньше, чем RustDesk его форвардит
(плюс RustDesk на Linux→Windows непредсказуемо шлёт именно Win-клавишу). Это НЕ влияет на работу.

**Можно без Super — всё остальное форвардится нормально:**
- Мышь (Пуск, иконки рабочего стола, таскбар, поиск) — работает; клик СНАЧАЛА внутрь удалённого экрана
  (дать фокус сессии), потом действия.
- Обычные клавиши/комбо: Ctrl+C / Ctrl+V / Ctrl+A, Alt+F4, Tab, Enter, стрелки, печать текста — доходят.
- «Выполнить» открывать НЕ через Win+R, а: клик по Пуску → набрать команду, либо двойной клик по ярлыку,
  либо поиск в таскбаре.
- Ctrl+Alt+Del / спец-клавиши при нужде — через тулбар самой RustDesk-сессии (иконки сверху окна).

Чинить (если когда-то реально понадобится Win): в RustDesk переключить режим клавиатуры (Map/Legacy) в тулбаре
сессии, либо убрать перехват Super у fluxbox. На практике не требуется — обходится мышью/Пуском.

### Грабли этого онбординга
- **`rustdesk --connect <ID>` молча выходит (exit 0) и не открывает окно**, если уже запущенный
  экземпляр RustDesk завис в несвежем состоянии (петля heartbeat/NAT). Лечится перезапуском
  нашего :99-клиента: `pkill -x rustdesk` (именно `-x`, чтобы не убить runuser-обёртки и свой
  bash), затем заново поднять `rustdesk --service` и `rustdesk` (фоновые задачи + dangerouslyDisableSandbox).
  После рестарта Connect/`--connect` сразу открывает окно сессии.
- В логе `~/.local/share/logs/RustDesk/rustdesk_rCURRENT.log` постоянно: `test nat: Failed to
  connect to …:21115` и `HTTP POST …:21114/api/heartbeat failed (IncompleteMessage)`. Это **НЕ
  фатально**: 21114 (API) у нас нет по дизайну, а NAT-проба к 21115 не мешает коннекту через
  relay 21117 (регистрация по 21116/UDP проходит — `update_pk` в логах сервера это подтверждает).
- Клик «Принять» через само окно RustDesk-сессии невозможно (ввод не форвардится до
  подтверждения) — принимаем либо у клиента, либо через параллельный AnyDesk.
